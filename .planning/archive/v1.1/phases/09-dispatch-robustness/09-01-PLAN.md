## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| registry policy -> service worker alarms | Static shared policy controls privileged SW timeout scheduling. |
| content adapter -> service worker response | Untrusted/hung tab message response crosses into dispatch result handling. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01-01 | Denial of Service | `resolveAdapterTimeouts` | mitigate | Reject `dispatchTimeoutMs < 30000` in resolver/tests per D-112; never create sub-minimum alarm silently. |
| T-09-01-02 | Denial of Service | `withAdapterResponseTimeout` | mitigate | Scope `setTimeout` to a single `chrome.tabs.sendMessage` Promise.race per D-113 and clear timer in finally. |
| T-09-01-03 | Repudiation | timeout diagnostics | mitigate | Persist TIMEOUT with internal diagnostic message while user-facing error code remains `TIMEOUT` and `retriable: true` per D-114. |
| T-09-01-04 | Elevation of Privilege | registry edits | accept | Phase does not modify host permissions or scriptFile origins; static `<all_urls>` remains forbidden by CLAUDE.md. |
</threat_model>
