**Gathered:** 2026-05-07
**Status:** Ready for planning

<domain>
## Task Boundary

使用 `frontend-design:frontend-design` skill 对 Web2Chat 现有 UI（popup + options 两个 entrypoint）进行**视觉与微交互层面的产品级优化**，让产品摆脱 AI 生成代码常见的通用感（Tailwind 默认 slate+sky），呈现 Editorial / data-dense 的工程美学。

**不在范围内：**

- 修改任何 `data-testid`、ARIA 属性、ID

**测试契约：** 所有 `data-testid` 必须保留；所有现有 Vitest 单测、Playwright e2e、type-check、lint 必须维持通过；Phase 1-7 的视觉 e2e（如 `tests/e2e/capture.spec.ts`）需继续通过。

<decisions>
## Implementation Decisions

### 优化范围
- **决策：** popup + options 全量（两个 entrypoint 都改）
- **依据：** 共享 token 体系才能呈现统一设计语言；options 当前 hierarchy 偏弱，与 popup 一起改投入回报最高。
- **影响：** 改动量较大（约 12 个组件 + 2 个 style.css + 1 个全局 token 文件），但 token 化后维护成本低于双套体系。

### 风格方向
- **决策：** Editorial / data-dense — 工程美学，信息密度优先
- **核心特征：**
  - 等宽数字（`tabular-nums` / `font-mono`）用于时间戳、dispatchId、host 名等元数据
  - metadata 用徽章/chip 表达（圆角、低对比度背景、紧凑 padding）
  - 视觉层级靠 typography weight + dividers + spacing rhythm，不依赖大色块
  - 紧凑但有节奏的留白：`{4, 8, 12, 16, 20, 24, 32}` spacing scale
  - 排版有印刷感：title 用 tracking-tight + 较粗 weight；body 用 leading-relaxed
- **参考气质：** Linear、Vercel Dashboard、The New York Times 工程页、Stripe Docs
- **依据：** Web2Chat 是开发者/重度用户的效率工具（一键投递 web → IM/AI），用户认知模型偏数据，Editorial 风格符合心智。

### 改动深度
- **决策：** 视觉 + 微交互（保留布局与 DOM 结构）
- **包含：**
  - 全新 token：色板（含 brand accent + neutral scale）、type scale、radii、shadow、duration、easing
  - 关键交互的 transition：focus、hover、state-change（loading → success、SendForm → InProgressView）
  - 新元数据呈现：URL host 用 chip、createdAt 用 monospace + 微弱 baseline、dispatchId 在 InProgressView 用 inline pill
  - 错误/空/加载/成功 4 态视觉差异化：用边线/留白节奏区分，不是堆叠图标
- **不包含：**
  - SendForm 字段重排或合并
  - CapturePreview 的字段折叠/分组
  - Options 三段卡片的合并或重组
  - DOM 节点新增/删除（仅 className/style 改动，必要时新增极薄包裹层用于动画）

### 依赖策略
- **决策：** 0 KB 新增依赖 — 纯 Tailwind v4 token + 自写 CSS（含 `@theme`, custom properties, `transition-*`）
- **依据：**
  - MV3 扩展 bundle size 敏感（Chrome Web Store + 扩展 cold-start）
  - Tailwind v4 已支持 `@theme` 自定义 token，足以表达整套设计系统
  - 微交互用 CSS `transition` + Tailwind `animate-*` 即可，无需 JS 动画库
- **影响：** 必须用 CSS variables 而非组件库；动画范围限定在 transition、不含 spring/layout-shift 类高级效果。

### Claude's Discretion
以下未在讨论中定义，由 Claude 在 Research/Plan 阶段决定：
- 具体色板（brand accent 颜色、neutral scale 步进）
- 具体字号 scale（基于 Editorial 风格的现代 type ramp）
- 具体 radius/shadow/border token
- 具体动画 duration / easing token
- 关键状态切换的微交互细节（如 InProgressView 进入动画、错误 banner 落入动画）
