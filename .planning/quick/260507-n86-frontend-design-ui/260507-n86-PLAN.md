# Quick Task 260507-n86: PLAN

## 任务总览

按 Editorial / data-dense 方向，对 Web2Chat popup + options 进行视觉与微交互全量重塑：抛弃通用 Tailwind slate+sky cliché，建立 stone+rust 暖色调编辑系统，serif display + sans body + mono meta 三层字型差异，0 KB 新增依赖。同时产出符合 awesome-design-system 格式的项目级 DESIGN.md。

#### 1.4 `DESIGN.md` (项目根)

完整 9 节文档，按 awesome-design-system Linear/Vercel/Claude 格式。包含：

1. **Visual Theme & Atmosphere** — 描述 Editorial / data-dense 哲学，stone+rust 暖色调，serif/sans/mono 三层字型，"page-as-print-layout"隐喻
2. **Color Palette & Roles** — 每个 token 的精确十六进制 + 用途说明 + dark mode 等价物
3. **Typography Rules** — 字体栈说明、type scale 表（display/title/heading/body/meta/label/micro 7 级）、UPPERCASE label 原则
4. **Component Stylings** — 6 类组件细节：
   - Buttons (Confirm primary / Cancel outlined / Destructive / Settings gear)
   - Inputs (Combobox / Textarea / Select — 全部 bottom-border editorial style)
   - Cards (edge-line, no fill)
   - Badges & Pills (host chip, dispatch ID pill, platform chip)
   - Banners (margin-note style for errors)
   - Modals (ConfirmDialog with scale-in)
5. **Layout Principles** — popup 360px 与 options 720px 容器；spacing scale {4, 8, 12, 16, 20, 24, 32}；hairline rule 分隔
6. **Depth & Elevation** — 边线优先、阴影克制；3 层（flat / edge / pop）
7. **Do's and Don'ts** — 关键规则：用 stone 不用 slate；用 serif 显示标题；UPPERCASE labels；底边线 input；rust 不 sky
8. **Responsive Behavior** — popup 360px 固定，options 720px 上限；handheld n/a（扩展不在移动端）
9. **Agent Prompt Guide** — quick reference + 5 个示例 prompt（如 "Build a SendForm field with editorial label", "Create an InProgressView spinner with stagger reveal" 等）

文档使用 Markdown 表格列出 token 与组件参数，便于查阅。
