# Quick Task 260617-225858 — 完成

把 `apps/marketing` 宣传页通过 GitHub Actions 发布到 GitHub Pages（项目子路径 `https://seven-steven.github.io/web2chat/`），仅 push main + 手动触发。

## 改动

| 文件 | 性质 | 说明 |
|---|---|---|
| `apps/marketing/vite.config.ts` | +3 行 | 顶层加 `base: process.env.VITE_BASE ?? '/'`，本地/CI 默认 base `/` 不变，部署 job 注入 `/web2chat/` |
| `.github/workflows/deploy-pages.yml` | 新建 | 双 job（build → deploy），官方 actions，`paths` 过滤仅 marketing 相关文件变动时触发 |

## 关键设计决策

**base path 经环境变量注入，不硬编码、不改 verify-build.mjs。**

- 默认链路（本地 dev、CI `ci.yml` 的 `site:build`+`site:verify`）：base `/`，favicon `/favicon.svg`，`verify-build.mjs:140` 的正则匹配不变 → **主验证链路零回归**。
- 部署链路：`VITE_BASE=/web2chat/ pnpm site:build`，产物 favicon `/web2chat/favicon.svg`、assets `/web2chat/assets/...`，与子路径自洽。部署产物不经 `site:verify`，两条路径互不干扰。

## 验证结果（本地实测）

- `pnpm site:build && pnpm site:verify` → OK，默认产物 favicon `/favicon.svg` ✓
- `VITE_BASE=/web2chat/ pnpm site:build` → 产物 favicon `/web2chat/favicon.svg`、assets 带前缀 ✓
- `pnpm typecheck` → EXIT=0 ✓
- `pnpm exec eslint apps/marketing/vite.config.ts` → EXIT=0 ✓
- workflow YAML 语法校验通过（jobs: build, deploy；on: push, workflow_dispatch）✓

## deploy-pages.yml 要点

- `on.push.branches: [main]` + `paths` 过滤（`apps/marketing/**`、workflow 自身、`pnpm-lock.yaml`）+ `workflow_dispatch`。
- `concurrency: { group: pages, cancel-in-progress: true }` — 避免并发部署。
- `build` job：checkout + pnpm@20 + `VITE_BASE=/web2chat/ pnpm site:build` + `configure-pages@v5` + `upload-pages-artifact@v3`（path: `apps/marketing/dist`）。
- `deploy` job：`needs: build`，`environment: github-pages`，`permissions: { pages: write, id-token: write }`，`deploy-pages@v4`。

## ⚠️ 需用户一次性配置（代码无法代劳）

首次部署前，在 GitHub 仓库：

**Settings → Pages → Build and deployment → Source 选 "GitHub Actions"**

未设置时 deploy job 会报环境错误，这是预期。设置后 push 到 main（改 marketing 文件）即自动部署到 `https://seven-steven.github.io/web2chat/`。

## 边界

- 不触碰扩展主链路代码、host_permissions、manifest。
- upload-pages-artifact 当前 pin `@v3`（与 `configure-pages@v5` / `deploy-pages@v4` 同为当前官方主版本）。
- 诊断面板出现的 marketing 模块解析错误是 IDE LSP 跨 workspace 路径别名的误报，真实 `pnpm typecheck` EXIT=0，与本改动无关。
