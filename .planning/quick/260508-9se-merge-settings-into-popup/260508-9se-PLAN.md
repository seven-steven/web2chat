# Plan: Merge Settings into Popup

## Goal

将当前独立的 options 页面（Language / Reset / GrantedOrigins 三节）合并到 popup 内，通过 PopupChrome 齿轮图标切换视图，无需跳转独立页面。

## Approach

齿轮图标作为视图切换器：send 模式 ↔ settings 模式。Settings 模式下渲染三个现有 section 组件。Options 页面保留但非主要入口。

## Tasks

### Task 1: Add view mode toggle to PopupChrome

**files:**
- `entrypoints/popup/components/PopupChrome.tsx`
- `entrypoints/popup/App.tsx`

**action:**
1. PopupChrome props 增加 `showSettings: boolean` 和 `onToggleSettings: () => void`
2. 齿轮图标在 showSettings=true 时旋转 45deg 并变 accent 色，点击关闭 settings
3. App.tsx 新增 `viewModeSig = signal<'send' | 'settings'>('send')`
4. 传 showSettings/onToggleSettings 给 PopupChrome

**verify:** 齿轮点击切换 viewModeSig，齿轮视觉反馈正确

**done:** 齿轮可点击切换模式

### Task 2: Render settings sections in popup

**files:**
- `entrypoints/popup/App.tsx`

**action:**
1. 从 options/components 导入 LanguageSection、ResetSection、GrantedOriginsSection
2. viewModeSig === 'settings' 时，渲染一个 scrollable settings 容器（max-height 约束 popup 可视区域），包含三个 section
3. 保持各 section 原有 data-testid 属性

**verify:** 切换到 settings 模式后三个 section 正确渲染

**done:** Settings 视图在 popup 内完整显示

### Task 3: Remove openOptionsPage call, adjust options page to redirect

**files:**
- `entrypoints/popup/components/PopupChrome.tsx`
- `entrypoints/options/App.tsx`

**action:**
1. 删除 PopupChrome 中 `chrome.runtime.openOptionsPage()` 调用
2. Options 页面保留（WXT manifest 需要），但添加提示文案引导用户通过 popup 齿轮图标访问设置

**verify:** 齿轮不再打开新页面

**done:** 齿轮仅做 inline toggle
