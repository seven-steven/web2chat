    **Key behavioral change**: After clicking Confirm, the popup may close because:
    1. chrome.permissions.request shows a dialog (steals focus) — but in dev mode this
       auto-grants without dialog, so popup survives
    2. chrome.tabs.create opens a new tab — but dispatch.start is called BEFORE tab
       creation, and window.close() only happens after dispatch.start returns Ok

    In dev mode with auto-grant permissions, the popup flow is:
    Confirm click -> permissions auto-grant -> dispatch.start sent -> SW creates tab ->
    popup window.close() (because dispatch.start returned Ok) -> new tab appears

    The E2E specs need to handle both the "popup closes" and "popup survives" scenarios.
    Since the popup calls window.close() after dispatch.start returns Ok, the popup is
    gone by the time the target page loads. The tests should NOT expect the popup to
    remain visible.

    **openclaw-dispatch.spec.ts** — Update both tests:

    Test 1 "happy path": After clicking Confirm, the popup closes (window.close).
    Wait for the new tab to appear via context.waitForEvent('page'), then verify
    the message list appears on the OpenClaw page. Do NOT try to interact with the
    popup after Confirm click.

    ```typescript
    test('openclaw dispatch: happy path — Confirm -> message appears in OpenClaw', async ({
      context,
      extensionId,
    }) => {
      const { articlePage, popup } = await openArticleAndPopup(context, extensionId);

      const sendToInput = popup.locator('[data-testid="combobox-popup-field-sendTo"]');
      await sendToInput.fill(OPENCLAW_URL);
      await popup.waitForTimeout(300);

      const confirm = popup.locator('[data-testid="popup-confirm"]');
      await expect(confirm).toBeEnabled({ timeout: 2_000 });

      // Click Confirm — popup calls window.close() after dispatch.start succeeds,
      // new tab opens with OpenClaw target page.
      const newPagePromise = context.waitForEvent('page', { timeout: 10_000 });
      await confirm.click();

      // Wait for OpenClaw page to open (new tab created by SW dispatch-pipeline)
      const openclawPage = await newPagePromise;
      await openclawPage.waitForLoadState('domcontentloaded');

      // Verify the adapter injected and message was composed/sent.
      // The adapter waits for textarea, composes markdown, sends via Enter,
      // then confirms via MutationObserver.
      const messageList = openclawPage.locator('.agent-chat__messages, [data-testid="message-list"]');
      await expect(messageList).toBeVisible({ timeout: 10_000 });

      await articlePage.close();
      await openclawPage.close();
    });
    ```

    Test 2 "full chain completes within 10s": Same pattern — time from Confirm
    click to message-list visible. Update timeout assertion to 10s (the 5s target
    is for the adapter chain, but tab creation + page load adds overhead).

    **openclaw-offline.spec.ts** — Update to handle popup reopen:

    The popup closes after Confirm (because dispatch.start returns Ok and popup calls
    window.close). The SW then opens the target tab, injects adapter, adapter fails
    with OPENCLAW_OFFLINE, writes error to storage.session.

    Test must reopen popup to see the error:

    ```typescript
    test('openclaw offline: non-OpenClaw chat page -> OPENCLAW_OFFLINE error in popup', async ({
      context,
      extensionId,
    }) => {
      const { articlePage, popup, popupUrl } = await openArticleAndPopup(context, extensionId);

      const sendToInput = popup.locator('[data-testid="combobox-popup-field-sendTo"]');
      await sendToInput.fill(OPENCLAW_OFFLINE_URL);
      await popup.waitForTimeout(300);

      const confirm = popup.locator('[data-testid="popup-confirm"]');
      await expect(confirm).toBeEnabled({ timeout: 2_000 });

      // Click Confirm — popup closes, SW opens target tab, adapter detects offline
      await confirm.click();

      // Wait for dispatch to complete (adapter injection + canDispatch failure)
      // The SW writes OPENCLAW_OFFLINE error to storage.session
      await popup.waitForTimeout(3_000).catch(() => {});
      // popup may already be closed, that's fine

      // Reopen popup — dispatch error should be visible
      const popup2 = await context.newPage();
      await articlePage.bringToFront();
      await popup2.goto(popupUrl);

      // Wait for popup to render (SendForm or loading — dispatch error shown via banner)
      await popup2.waitForSelector(
        '[data-testid="popup-sendform"], [data-testid="popup-loading"]',
        { timeout: 10_000 },
      );

      // The dispatch error (OPENCLAW_OFFLINE) should be shown as error banner
      const errorBanner = popup2.locator('[data-testid="error-banner-OPENCLAW_OFFLINE"]');
      await expect(errorBanner).toBeVisible({ timeout: 15_000 });

      await popup2.close();
    });
    ```

    Key change: After Confirm click, don't try to interact with the original popup.
    Instead, open a new popup page and check for the error banner.
