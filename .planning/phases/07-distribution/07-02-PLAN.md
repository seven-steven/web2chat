Content requirements (verbatim from D-89 + CLAUDE.md + DST-02):

**Data We Collect** — enumerate all 6 fields:
- `url` — the URL of the page you clip
- `title` — the page title
- `description` — the page description (from meta tags or Readability excerpt)
- `content` — the page content (extracted by Readability, sanitized by DOMPurify, converted to Markdown by Turndown)
- `prompt` — your user-defined prompt text
- `create_at` — timestamp of when you initiated the clip

**How Data Is Stored**:
- All data is stored exclusively in `chrome.storage.local` (persistent) and `chrome.storage.session` (temporary, cleared when browser closes)
- No data is stored on any remote server
- No data is synced to the cloud

**How Data Is Transmitted**:
- Data is transmitted ONLY when you explicitly confirm a dispatch to a target IM/AI chat session
- Transmission occurs through browser tab navigation — the extension opens the target page and injects your composed message via DOM interaction
- No data is sent to any server, API endpoint, or third-party service at any time

**Data We Do NOT Collect** (explicit negation list):
- We do not operate or communicate with any remote server
- We do not use any third-party analytics, tracking, or telemetry SDK
- We do not store or request any API keys
- We do not use cookies or web beacons
- We do not serve advertisements or use ad tracking
- We do not sync data to the cloud or any external storage
- We do not collect browsing history beyond the current active tab when you click the extension icon

**Third-Party Services**: Web2Chat does not integrate with any third-party services for data collection, analytics, or advertising.

**Your Control Over Data**:
- You can clear all stored data via the extension's Settings page ("Reset all history")
- You can revoke granted permissions for specific origins via the Settings page
- Uninstalling the extension removes all stored data

**Changes to This Policy**: reference that updates will be committed to this file in the repository.

**Contact**: reference the GitHub repository issues page.
