## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline metadata separators, compact status-label gap |
| sm | 8px | Compact element spacing, badge/chip internal gap, table cell micro spacing |
| md | 16px | Default element spacing, paragraph-to-control spacing, card internal padding |
| lg | 24px | Section block padding, grouped claim matrix blocks, trust/limits card padding |
| xl | 32px | Layout gaps between major content groups: Hero preview, use-case row, platform module |
| 2xl | 48px | Major section breaks in the Phase 15 landing page outline |
| 3xl | 64px | Page-level spacing between Hero/use cases/payload/platform/trust/CTA bands |

Exceptions: 44px minimum touch target for primary CTA, secondary CTA, and icon-only links in Phase 15; 1px borders/rules allowed for editorial/data-dense separators; 2px focus outline allowed using accent ring.

**Source notes:** default 4/8-point scale from UI researcher contract; visual density aligned with `13-CONTEXT.md` D-02 compact first-screen requirement and existing editorial/data-dense token direction.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

**Typography contract:**
- Use exactly two weights: regular 400 and semibold 600.
- Use exactly four sizes: 14px, 16px, 20px, 28px.
- Structured-payload preview and claims matrix snippets use mono font at Label or Body size only; do not introduce a fifth size.
- Page copy should remain concise: Hero one-line value statement, then supporting modules. Phase 13 writes guardrails, not complete long-form bilingual page copy.

**Source notes:** D-02 requires compact first-screen density; D-14 requires copy guardrails rather than final long copy; system font stack detected in `_shared-tokens.css`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `#fafafa`; dark `#1e1e1e` | Page background, large canvas bands, planning diagrams/background surfaces |
| Secondary (30%) | Light `#ffffff` / `#f4f4f5`; dark `#262626` / `#2a2a2a` | Cards, structured-payload preview, claims matrix blocks, platform/trust/limits panels |
| Accent (10%) | Light `#10b981`; dark `#34d399` | Primary CTA, current-step marker in three-step flow, verified/source-backed claim tags, focus ring, active link underline |
| Destructive | Light `#dc2626`; dark `#f87171` | Destructive actions only; Phase 13 has no destructive user action |

Accent reserved for: primary CTA, focus-visible ring, structured-payload “send/go” arrow, verified/source-backed status tag, active section anchor, and one current-step marker in the three-step flow. Do not use accent for every link, all platform badges, or decorative gradients.

**Semantic status colors:**
- Warning: existing token `--color-warn` for `live UAT pending`, `known risk`, or `placeholder` asset labels.
- Destructive/danger: only for true destructive actions or forbidden wording markers in internal planning artifacts; not for normal public-facing risk disclosures.

**Source notes:** Existing token source is `entrypoints/_shared-tokens.css`; Phase 13 requirements require trust boundaries and known-risk clarity without turning risks into marketing proof.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `查看项目源码` |
| Empty state heading | `尚未定义页面 claims` |
| Empty state body | `先创建 claims matrix，再为 Hero、use cases、payload、platforms、trust、limits、CTA 逐条绑定事实源。` |
| Error state | `Claim 缺少事实源或验证说明。补充 source file / section 与 verification note 后再进入页面实现。` |
| Destructive confirmation | none — Phase 13 不定义任何 destructive action；不得设计删除 claims/source 的确认流作为本 phase 范围 |

**Narrative contract for Phase 15 derived from Phase 13:**
1. Hero: 一句话说明“当前网页结构化信息 + saved prompt → IM / AI Agent chat session”。
2. Use cases: personal knowledge capture、team sharing、Agent / llm-wiki workflows。
3. Structured-payload example: 必须展示 `title` / `url` / `description` / `create_at` / `content` + `prompt`，用来说明相对手动复制粘贴的差异。
4. Supported platforms: 主列表只展示 OpenClaw / Discord / Slack / Telegram；Telegram 必须带 `live UAT pending / known risk` 边界，不得写成 fully verified。
5. Trust section: 独立呈现 no server、no telemetry、local storage、direct browser delivery、production permission model。
6. Known limits: Feishu/Lark 只在 limits/roadmap 语境说明 evaluated and dropped due to unreliable shared URL targeting；不得出现在支持平台徽章中。
7. CTA: CTA-01/CTA-02 后续可分化为 source repository 与 Chrome/Chromium installation path；Phase 13 默认主 CTA 为 `查看项目源码`。

**Public claim wording rules:**
- Allowed wording 必须可追溯到 `.planning/PROJECT.md`、`PRIVACY.md`、`STORE-LISTING.md` copy style、或 production `wxt.config.ts`。
- Forbidden wording 必须显式列入 claims matrix，至少覆盖：`fully verified Telegram`、`Feishu/Lark supported`、production `tabs` permission、static production `<all_urls>` host permission、任何 telemetry/analytics/server overclaim。
- Phase 13 不写完整中英页面长文，只写 page outline、claims matrix、copy guardrails、asset status rules、maintenance rules。

**Source notes:** CTA inferred from CTA-01 repository/project entry requirement and Phase 13 being pre-installation planning; empty/error states apply to planning artifact validation, not runtime UI.
