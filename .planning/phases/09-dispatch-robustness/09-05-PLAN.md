<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service worker warning record -> popup UI | Actionable selector warning controls whether user can initiate second dispatch. |
| popup UI -> dispatch.start | User confirmation sends captured page data to external IM page. |
| locale files -> rendered popup copy | User-visible warning/error text is rendered in the extension popup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-05-01 | Information Disclosure | low-confidence confirm | mitigate | Require explicit user confirm before second dispatch and keep cancel path no-send per D-123/D-124. |
| T-09-05-02 | Tampering | selectorConfirmation persistence | mitigate | One-shot flag is only in fresh dispatch.start input; tests grep no writes to history/binding/draft per D-126. |
| T-09-05-03 | Spoofing | dialog accessibility/focus | mitigate | role=dialog, aria-modal, focus trap, Escape/overlay cancel make state clear to keyboard/screen-reader users. |
| T-09-05-04 | Information Disclosure | popup copy | mitigate | All text uses i18n keys; no raw adapter/SW message rendered. |
</threat_model>
