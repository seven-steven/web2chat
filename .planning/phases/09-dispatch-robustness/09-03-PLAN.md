<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service worker/storage -> popup UI | Error records and immediate dispatch failures become user-visible UI state. |
| user form -> fresh dispatch.start | Retry sends current edited content to target IM after user action. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03-01 | Information Disclosure | `ErrorBanner` | mitigate | Continue rendering only i18n text by ErrorCode; never raw `error.message` per UI-SPEC. |
| T-09-03-02 | Denial of Service | retry action | mitigate | Retry is user-clicked and creates one fresh dispatchId; no automatic retry queue per D-119/D-121. |
| T-09-03-03 | Repudiation | old failed record | mitigate | Clear old active/error before new active dispatch per D-122 while retaining old session record only for diagnostics. |
| T-09-03-04 | Tampering | stale payload reuse | mitigate | Rebuild payload from current form values on Retry per D-119; test edited title/description/content. |
</threat_model>
