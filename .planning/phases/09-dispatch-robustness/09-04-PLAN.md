<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| target page DOM -> content adapter | Untrusted/changed third-party DOM determines selector tier. |
| content adapter -> service worker | Adapter response warning crosses from tab context to privileged dispatch storage. |
| service worker -> popup | Warning state prompts user action before sending captured content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04-01 | Information Disclosure | Discord selector tier3 | mitigate | Tier3 class-fragment match returns SELECTOR_LOW_CONFIDENCE warning before compose/send; no message sent per D-123. |
| T-09-04-02 | Tampering | adapter response warning | mitigate | Warning code is schema-literal validated; invalid warning strings rejected/default ignored. |
| T-09-04-03 | Repudiation | warning state | mitigate | Persist explicit warning/needs_confirmation record with dispatchId and warning code for popup resume/audit. |
| T-09-04-04 | Elevation of Privilege | ErrorCode namespace | mitigate | Do not add SELECTOR_LOW_CONFIDENCE to ErrorCode; keep failure handling separated per D-127. |
</threat_model>
