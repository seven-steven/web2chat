<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| content data → proof visuals | truth-safe 文案被包装成 mockup，若标签缺失会让演示误看成真实截图 |
| CTA style primitive → navigation components | 按钮组件将被 Hero 和底部 CTA 复用，视觉或语义错误会全页扩散 |
| decorative UI → accessibility tree | 纯装饰图形若进入可访问树，会制造错误语义和键盘噪音 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Spoofing | proof mockup components | mitigate | 为每个 proof 组件强制渲染可见 `mockup` 标签和 `source/status/version` 元数据，单测逐个断言 |
| T-15-05 | Tampering | stepper semantics | mitigate | stepper 只编码固定三步 `capture → target → send`，不接受任意未审核步骤数组作为默认 UI |
| T-15-06 | Denial of Service | keyboard/a11y flow | mitigate | CTA 按钮与 proof 组件装饰 SVG 全部用清晰焦点状态与 `aria-hidden` 装饰规则，避免无意义 tab 停留 |
| T-15-SC | Tampering | npm/pnpm installs | accept | 本计划不新增依赖、不运行 package-manager install，供应链面保持不变 |
</threat_model>
