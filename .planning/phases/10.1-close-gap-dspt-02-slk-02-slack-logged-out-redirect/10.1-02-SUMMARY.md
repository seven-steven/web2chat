## Accomplishments

- Added true RED coverage for the real live failure path: `executeScript` permission failures plus `sendMessage` channel-close/BFCache failures on Slack signin/workspace-signin redirects.
- Updated the dispatch pipeline to re-check `tab.url` before surfacing final transport errors, so Slack login redirects consistently map to `NOT_LOGGED_IN`.

## Decisions Made

- Reused the existing `isLoggedOutUrlForAdapter()` contract everywhere instead of adding Slack-specific branches to the pipeline.
- Treated Chrome transport strings containing `message channel is closed` / `back/forward cache` as remappable login-redirect failures rather than terminal execution failures.
- Preserved the original source snapshot from the failed dispatch record on popup reopen, rather than relying on a fresh capture from the current active tab.
- Kept the new real-site Playwright test opt-in via `SLACK_E2E_SESSION_URL`, matching existing local-only E2E conventions.

### Auto-fixed Issues

**1. [Rule 1 - Bug] Slack BFCache error string differed from the first closed-channel signature**
- **Found during:** Task 2 live Chrome/Playwright verification
- **Issue:** Service worker storage showed `EXECUTE_SCRIPT_FAILED` with `The page keeping the extension port is moved into back/forward cache, so the message channel is closed.` — this string was not matched by the first closed-channel regex.
- **Fix:** Extended the sendMessage failure regex to match `message channel is closed` and `back/forward cache`, then re-ran the real-session Playwright regression.
- **Files modified:** `background/dispatch-pipeline.ts`, `tests/unit/dispatch/login-detection.spec.ts`
- **Verification:** Focused unit regression passed and real Slack session rerun ended in `NOT_LOGGED_IN`.
- **Committed in:** pending commit in current working tree

**2. [Rule 1 - Bug] Popup reopen initially rendered loading + error banner before source snapshot restored**
- **Found during:** Playwright popup-reopen verification
- **Issue:** Even after preserving `failedDispatchSnapshot`, popup still timed out waiting for `popup-sendform` because the failed snapshot was only applied after `capture.run` completed.
- **Fix:** Restored `rec.snapshot` immediately when the active dispatch record is already in `error` state, before the fresh capture promise settles.
- **Files modified:** `entrypoints/popup/App.tsx`
- **Verification:** Real Slack session popup reopen now shows the original article title/source while also rendering the `NOT_LOGGED_IN` banner.
- **Committed in:** pending commit in current working tree
