    Modify `onTabComplete` in `background/dispatch-pipeline.ts` to detect Discord login redirects. The detection logic:

    After `tabs.onUpdated:complete` fires, query the tab's actual URL via `chrome.tabs.get(tabId)`. Compare the actual tab URL against the dispatch record's `send_to`:

    1. Find the adapter for `record.send_to` (this succeeds -- it's `discord` adapter)
    2. Check if the actual tab URL matches ANY of the adapter's `hostMatches` patterns (e.g., `https://discord.com/*`)
    3. If host matches but `findAdapter(actualTabUrl)` returns null or returns a DIFFERENT adapter, the tab navigated to a non-channel page on the same host (login redirect)
    4. In this case: `failDispatch(record, 'NOT_LOGGED_IN', 'Discord login redirect detected', true)` and skip adapter injection

    Concrete change to `onTabComplete`:

    ```typescript
    export async function onTabComplete(
      tabId: number,
      changeInfo: chrome.tabs.OnUpdatedInfo,
      _tab: chrome.tabs.Tab,
    ): Promise<void> {
      if (changeInfo.status !== 'complete') return;
      const all = await dispatchRepo.listAll();
      for (const record of all) {
        if (record.state !== 'awaiting_complete') continue;
        if (record.target_tab_id !== tabId) continue;
        const adapter = findAdapter(record.send_to);
        if (!adapter) {
          await failDispatch(record, 'INTERNAL', 'Adapter disappeared between opening and complete', false);
          continue;
        }

        // Phase 5 D-70: Login redirect detection for Discord (and future adapters).
        // After tab completes loading, check if the actual URL still matches the adapter.
        // If tab URL is on the adapter's host but doesn't match adapter.match(),
        // a redirect occurred (e.g., discord.com/login?redirect_to=...).
        if (adapter.hostMatches.length > 0) {
          let actualUrl: string | undefined;
          try {
            const tab = await chrome.tabs.get(tabId);
            actualUrl = tab.url;
          } catch {
            // Tab may have been closed
          }
          if (actualUrl && !adapter.match(actualUrl)) {
            // Tab URL is NOT matched by the adapter -- check if it's on the same host
            const isHostMatch = adapter.hostMatches.some((pattern) => {
              // Simple host suffix check: extract hostname from pattern and compare
              try {
                const patternHost = new URL(pattern.replace('*', 'x')).hostname;
                const actualHost = new URL(actualUrl!).hostname;
                return actualHost === patternHost || actualHost.endsWith('.' + patternHost);
              } catch { return false; }
            });
            if (isHostMatch) {
              await failDispatch(record, 'NOT_LOGGED_IN', `Redirected to ${actualUrl}`, true);
              continue;
            }
          }
        }

        await advanceToAdapterInjection(record, adapter.scriptFile);
      }
    }
    ```

    This approach is generic (works for any adapter with hostMatches) but primarily targets Discord's login redirect pattern per D-70. When the tab navigates to `discord.com/login`, the discord adapter's `match()` returns false (pathname doesn't start with `/channels/`), but the host is still `discord.com` (matching `hostMatches`), so the dispatch-pipeline returns NOT_LOGGED_IN.
