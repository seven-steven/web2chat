## E2E Routing Approach (RESOLVED)

The Discord adapter's `match()` requires `hostname === 'discord.com'`, but E2E stubs are served at `localhost:4321`. This means the full popup->SW pipeline cannot auto-discover the Discord adapter for stub URLs. The existing mock-platform tests already validate the full pipeline routing. Discord E2E tests focus on adapter-specific value:

**Direct adapter injection pattern:**
1. Navigate Playwright to `http://localhost:4321/discord/channels/0/12345` (fixture served at this path)
2. Use `chrome.scripting.executeScript` (via CDP/evaluate) to inject `content-scripts/discord.js` into the stub page
3. Send `ADAPTER_DISPATCH` message via `chrome.tabs.sendMessage`
4. Verify the adapter's DOM interaction (paste injection, Enter send, message appearance)

This is the same technique the dispatch-pipeline uses internally -- we just bypass the URL matching step.

**Stub URL format:** All stub URLs use `/discord/channels/<serverId>/<channelId>` path format (e.g., `http://localhost:4321/discord/channels/0/12345`). This ensures `extractChannelId()` in the adapter can parse the channelId from the URL pathname, producing correct channel mismatch detection in E2E tests. The fixture HTML is served for all paths under `/discord/` via a catch-all or symlink setup.

**Login detection test:** Uses the popup->SW pipeline with a real Discord URL that will 404/redirect on localhost. The dispatch-pipeline checks the actual tab URL after `tabs.onUpdated:complete`, detects host matches but adapter.match() fails, and returns NOT_LOGGED_IN.
