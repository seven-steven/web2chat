<locked_decisions>
- D1: SINGLE-JOB EXTENSION (RESEARCH Pattern 2). Do NOT add parallel jobs. Rationale: the 4 new steps add ~10-15s wall-clock; parallel jobs would each pay ~30-40s checkout + `pnpm install --frozen-lockfile`. Existing `cache: 'pnpm'` on setup-node@v6 already handles monorepo lockfile caching.
- D2: ORDER IS LOAD-BEARING (RESEARCH Pitfall 2 + PATTERNS.md ## CI step ordering invariant). The 4 new steps MUST come after `pnpm verify:manifest`, in order: `site:build → site:verify → verify:readme → verify:claims`. verify:manifest's first step is `wxt build` (package.json:27), which produces the prod manifest branch. verify:claims reads that manifest. If reordered, verify:claims would crash (manifest missing) or false-pass against a dev manifest.
- D3: No `secrets:` or `env:` additions — all four steps read only public repo files (manifest, locale JSON, PRIVACY.md, README.md, marketing source). The job's existing permission surface is unchanged.
- D4: No `timeout-minutes` change — the existing 10-minute budget absorbs ~15s of new work with margin.
</locked_decisions>
