<objective>
Feishu/Lark content script + selector 测试 + fixture HTML。

Purpose: Complete the feishu adapter with the full content script following the Telegram
pattern. The content script is injected by dispatch-pipeline via chrome.scripting.executeScript.
It registers a one-shot ADAPTER_DISPATCH listener that handles the complete dispatch flow:
login detection -> editor finding -> message formatting -> MAIN world paste -> send confirmation.

Send confirmation strategy (D-162 research conclusion): editor textContent clearance polling
(same as Telegram). After MAIN world paste + send trigger, poll editor.textContent every 300ms
for up to 5 attempts. If editor clears -> success. If not -> TIMEOUT.

Rate limit: 5s per URL (same as Telegram). Use the send_to URL as key (not chat ID extraction,
since feishu URL chat ID extraction is unreliable without verified URL structure).

Selector strategy (D-164): ARIA-first three-tier fallback:
  Tier 1: [contenteditable="true"][role="textbox"]
  Tier 2: .message-input [contenteditable="true"]
  Tier 3: [contenteditable="true"] (low confidence)

Output: Working content script + passing selector/dispatch tests + fixture HTML.
</objective>
