## Key Accomplishments

1. 完成适配器架构泛化，新增平台无需再改 dispatch pipeline 或 service worker 入口。
2. 完成投递鲁棒性加固，统一覆盖超时分层、登录态检测、可重试 UI 与低置信度确认流。
3. Slack 适配器 shipped，覆盖 URL 匹配、登录检测、富文本注入、发送确认与中英文本地化。
4. Slack logged-out redirect 历史 blocker 已闭环，真实回归确认 `NOT_LOGGED_IN` 语义与 popup 原始 snapshot 保留。
5. Telegram 适配器 shipped，覆盖 Web K URL、contenteditable 注入、4096 字截断、发送确认与 i18n。
6. Feishu/Lark 已完成实现验证，但因共享 URL blocker 被明确移出最终 shipped scope。

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Registry-driven adapter architecture | 让新增平台只改适配器层，不动 pipeline/SW | ✓ Phase 8-12 持续验证成立 |
| 登录态判断泛化为声明式策略 | 统一处理不同平台登录页与跳转差异 | ✓ Slack / Telegram 共用同一调度框架 |
| 低置信度选择器改为用户确认流 | 避免误投递，同时不直接阻断操作 | ✓ popup confirmation + retry 形成闭环 |
| Slack redirect host 单独声明可观测权限 | 只为 `slack.com` redirect 增加最小权限面 | ✓ logged-out redirect regression 已关闭 |
| Feishu/Lark 从 shipped scope 移除 | 当前 URL 模型无法可靠定位聊天目标 | ✓ v1.1 shipped scope 保持稳定可信 |
