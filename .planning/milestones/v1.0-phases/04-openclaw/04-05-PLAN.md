<objective>
Fix all 4 UAT gaps caused by popup closing when chrome.permissions.request or
chrome.tabs.create fires during dispatch to dynamic-permission adapters (OpenClaw).

Root cause: SendForm.handleConfirm calls chrome.permissions.request in the popup
click handler. The permission dialog steals focus, closing the popup. All code
after that call (grantedOriginsRepo.add, sendMessage('dispatch.start')) never
executes. On popup reopen, the dispatch intent is lost.

Fix strategy:
1. **Intent persistence**: Before chrome.permissions.request, save the full
   DispatchStartInput to a new storage.local item. If the popup closes, the
   intent survives.
2. **Popup resume on reopen**: App.tsx mount checks for a saved pending dispatch.
   If found and permission is now granted -> auto-send dispatch.start. If
   permission denied -> show OPENCLAW_PERMISSION_DENIED.
3. **grantedOriginsRepo.has() fix**: Query chrome.permissions.contains as the
   authoritative source, not just the local mirror. This prevents the infinite
   loop where has() returns false even though Chrome granted the permission.
4. **Dispatch error priority**: When popup reopens with an active dispatch error
   in storage.session, show it instead of letting capture errors (from the
   now-current error page tab) take precedence.

Purpose: Make the OpenClaw dispatch flow resilient to Chrome's popup-closes-on-
focus-loss behavior. Same fix covers gaps 1-3; gap 4 (E2E) follows in plan 06.
Output: Popup that survives permission dialogs and tab switches for OpenClaw.
</objective>

    **Step 3: Fix grantedOriginsRepo.has() to use chrome.permissions.contains**

    In `shared/storage/repos/grantedOrigins.ts`, modify `has()`:

    ```typescript
    export async function has(origin: string): Promise<boolean>: Promise<boolean> {
      // Authoritative source: Chrome's permission API. The local mirror may be stale
      // if popup closed between chrome.permissions.request and grantedOriginsRepo.add().
      const chromeGranted = await chrome.permissions.contains({
        origins: [origin + '/*'],
      });
      if (chromeGranted) return true;
      // Fallback: local mirror (for environments where chrome.permissions not available)
      const current = await grantedOriginsItem.getValue();
      return current.includes(origin);
    }
    ```

    The chrome.permissions.contains call is the authoritative check. The local mirror
    is only a fallback. This prevents the infinite loop where:
    1. User grants permission (Chrome grants it)
    2. Popup closes before grantedOriginsRepo.add() executes
    3. Next popup open: has() checks local mirror -> false -> requests permission again
    4. Popup closes again -> infinite loop

    **Step 4: Rewrite SendForm.handleConfirm for intent-first flow**

    In `entrypoints/popup/components/SendForm.tsx`, rewrite the `handleConfirm` function:

    ```typescript
    async function handleConfirm() {
      if (submitting) return;
      setSubmitting(true);
      const dispatchId = crypto.randomUUID();
      const input: DispatchStartInput = {
        dispatchId,
        send_to: props.sendTo,
        prompt: props.prompt,
        snapshot: {
          ...props.snapshot,
          title: props.titleValue,
          description: props.descriptionValue,
          content: props.contentValue,
        },
      };

      const adapter = findAdapter(props.sendTo);
      if (adapter && adapter.hostMatches.length === 0) {
        // Dynamic permission adapter (openclaw)
        const targetOrigin = new URL(props.sendTo).origin;
        const alreadyGranted = await chrome.permissions.contains({
          origins: [targetOrigin + '/*'],
        });

        if (!alreadyGranted) {
          // Save dispatch intent BEFORE requesting permission.
          // If popup closes during the permission dialog, the intent survives.
          await draftRepo.savePendingDispatch(input);
          const granted = await chrome.permissions.request({
            origins: [targetOrigin + '/*'],
          });

          if (!granted) {
            // Permission denied — popup may or may not have closed.
            // If popup survived (auto-grant scenario where granted=false shouldn't happen,
            // but defensive): show error.
            setSubmitting(false);
            await draftRepo.clearPendingDispatch();
            props.onDispatchError('OPENCLAW_PERMISSION_DENIED', targetOrigin);
            return;
          }

          // Permission granted. Update local mirror (best-effort; popup may close).
          await grantedOriginsRepo.add(targetOrigin).catch(() => {});
          // Clear pending intent — dispatch will proceed.
          await draftRepo.clearPendingDispatch();
        }
      }

      // Notify App so it can render InProgressView immediately
      props.onConfirm(dispatchId);
      try {
        const res = await sendMessage('dispatch.start', input);
        if (res.ok) {
          window.close();
        } else {
          setSubmitting(false);
          props.onDispatchError(res.code, res.message);
        }
      } catch (err) {
        setSubmitting(false);
        const msg = err instanceof Error ? err.message : String(err);
        props.onDispatchError('INTERNAL', msg);
      }
    }
    ```

    Key changes from current code:
    - Uses `chrome.permissions.contains` instead of `grantedOriginsRepo.has()` (authoritative)
    - Saves `DispatchStartInput` to storage BEFORE `chrome.permissions.request`
    - Clears pending dispatch on both grant and deny paths
    - `grantedOriginsRepo.add` is best-effort (`.catch(() => {})`) since it's now non-critical
