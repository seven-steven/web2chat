# web2chat 隐私政策

**最后更新：** 2026-05-07

## 我们收集的数据

当您使用 web2chat 剪辑网页时，我们会从当前活动的浏览器标签页中处理以下数据字段：

- `url` -- 您所剪辑页面的 URL
- `title` -- 页面标题
- `description` -- 页面描述（从 meta 标签或 Readability 生成的摘要中提取）
- `content` -- 页面内容（由 Readability 提取，经 DOMPurify 净化，并由 Turndown 转换为 Markdown）
- `prompt` -- 您自定义的提示文本
- `create_at` -- 您发起剪辑时的时间戳

我们**仅在您点击扩展图标时**收集这些数据。我们不会在后台被动收集任何数据。

## 数据存储方式

所有数据仅存储在您的本地设备上，使用浏览器存储 API：

- `chrome.storage.local` -- 持久存储，跨浏览器会话保留数据
- `chrome.storage.session` -- 临时存储，在浏览器关闭时清除

没有任何数据存储在任何远程服务器上。没有任何数据同步到云端。

## 数据传输方式

数据**仅在您明确确认向目标 IM 或 AI 聊天会话投递时**才会传输。传输过程如下：

1. 您选择目标聊天会话 URL 并确认投递
2. 扩展在浏览器标签页中打开或激活目标页面
3. 扩展通过 DOM 交互将您组合的消息注入目标页面

在任何时候，数据都不会被发送到任何服务器、API 端点或第三方服务。所有传输都通过直接的浏览器标签页导航完成。

## 我们不会收集的数据

我们明确声明 web2chat **不会**做以下事情：

- 我们不会运营或与任何远程服务器通信
- 我们不会使用任何第三方分析、追踪或遥测 SDK
- 我们不会存储或请求任何 API 密钥
- 我们不会使用 Cookie 或网络信标
- 我们不会投放广告或使用广告追踪
- 我们不会将数据同步到云端或任何外部存储
- 我们不会在您点击扩展图标时之外收集浏览历史

## 第三方服务

web2chat 不集成任何第三方服务进行数据收集、分析或广告投放。本扩展完全在您的浏览器内运行。

## 您对数据的控制

您对 web2chat 存储的所有数据拥有完全控制权：

- 您可以通过扩展的设置页面清除所有存储数据（"重置所有历史"）
- 您可以通过设置页面撤销已授予的特定来源权限
- 卸载扩展将立即删除所有存储数据

## 政策变更

如果本隐私政策发生变更，更新将反映在本文件中。完整的修订历史可在公开的 GitHub 仓库中查看。

## 联系方式

如果您对本隐私政策有任何疑问或顾虑，请在 [web2chat GitHub 仓库](https://github.com/nicholaschenai/web2chat)提交 issue。
