# web2chat 隐私政策

**最后更新：** 2026-06-18

web2chat 是一款本地优先的 Chrome MV3 网页剪藏扩展。它帮助您抓取当前网页的结构化信息，并结合您自定义的 prompt，发送到您选择的受支持聊天或 AI Agent Web 会话。

web2chat 不运营自有后端服务，不接收、不存储、不出售、也不分析您的剪藏网页内容。

## 我们处理的数据

当您使用 web2chat 剪藏网页时，我们会从当前活动的浏览器标签页中处理以下数据字段：

- `url` -- 您所剪藏页面的 URL
- `title` -- 页面标题
- `description` -- 页面描述，从 meta 标签或 Readability 生成的摘要中提取
- `content` -- 可读页面内容，由 Readability 提取，经 DOMPurify 净化，并由 Turndown 转换为 Markdown
- `prompt` -- 您自定义的提示文本
- `create_at` -- 您发起剪藏时的时间戳
- `send_to` / 目标 URL -- 您选择作为投递目的地的聊天或 AI Agent 会话 URL
- 本地历史与绑定 -- 用于提高重复投递效率的目标历史、prompt 历史、prompt 与目标绑定关系
- 投递状态 -- 用于完成或重试投递流程的临时状态

我们仅在您明确点击扩展后处理页面内容。我们不会在后台被动收集数据。

## 数据存储方式

数据使用 Chrome 扩展存储 API 保存在您的本地设备上：

- `chrome.storage.local` -- 持久本地存储，用于保存设置、目标历史、prompt 历史、prompt 绑定、已授权 origin 记录
- `chrome.storage.session` -- 临时会话存储，用于保存投递状态和进行中的流程数据

剪藏的网页内容、prompt、目标历史和设置不会存储到 web2chat 远程服务器。web2chat 不会将这些数据同步到云端。

## 数据传输方式

数据仅在您明确确认向目标 IM 或 AI 聊天会话投递时才会传输。投递过程如下：

1. 您选择或输入目标聊天会话 URL，并确认投递。
2. 扩展在浏览器标签页中打开或激活目标页面。
3. 扩展通过 DOM 交互将组合后的消息插入目标页面。
4. 目标服务收到该消息，因为您选择将内容发送到那里。

web2chat 不会把剪藏内容发送到开发者运营的服务器或 API 端点。投递通过您的本地浏览器标签页和您选择的目标服务完成。

## 第三方服务

web2chat 可以向受支持的第三方或自托管目标投递内容，目前包括 OpenClaw Web UI、Discord Web、Slack Web 和 Telegram Web。

当您选择将内容发送到第三方聊天平台时，该平台可能会依据其自身服务条款和隐私政策处理该消息。web2chat 不控制这些第三方服务。

web2chat 不集成任何第三方分析、追踪、遥测或广告服务。

## 我们不会收集或使用的数据

web2chat 不会：

- 为剪藏内容运营或通信到开发者控制的后端服务
- 在您与扩展交互前收集数据
- 读取您的完整浏览历史
- 从目标平台读取您的聊天历史
- 存储或请求 API 密钥
- 使用 Cookie、网络信标、分析 SDK、遥测 SDK 或广告 SDK
- 出售用户数据
- 将用户数据转让给数据经纪商
- 将用户数据用于广告或个性化广告
- 将用户数据用于信用评估或贷款目的
- 将剪藏数据同步到云端存储

## 权限

web2chat 仅请求实现其单一用途所需的权限：

- `activeTab` -- 在用户主动操作后访问当前活动标签页，用于提取页面标题、URL、摘要和可读正文
- `scripting` -- 运行页面抽取脚本，并向用户选择的目标聊天网页注入对应平台的发送适配器
- `storage` -- 在本地保存设置、目标历史、prompt 历史、prompt 绑定、已授权 origin 和投递状态
- `webNavigation` -- 观察受支持目标页面的加载与 SPA 导航状态，使投递流程能在正确时机继续
- `alarms` -- 管理投递流程的超时计时器
- `host_permissions` -- 当用户选择向受支持公共平台投递时，访问这些目标平台域名
- `optional_host_permissions` -- 仅在用户配置特定自部署 OpenClaw 或自定义目标 origin 时，于运行时请求该具体 origin 的访问权限

扩展默认不会获得访问所有网站的权限。可选 host 权限仅在需要时针对具体 origin 请求。

## 您对数据的控制

您可以控制 web2chat 本地保存的数据：

- 您可以通过扩展设置页面清除历史和绑定关系。
- 您可以通过扩展设置页面或 Chrome 扩展设置撤销已授权的 origin 权限。
- 您可以通过卸载扩展删除其本地扩展存储。

## 有限使用承诺

web2chat 仅将处理的数据用于提供或改进其单一用途：帮助您把当前网页信息和您的 prompt 发送到您选择的受支持聊天或 AI Agent 会话。

web2chat 遵守 Chrome Web Store 用户数据政策，包括 Limited Use 要求。我们不会出售、转让或将用户数据用于无关目的。
